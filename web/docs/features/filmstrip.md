# Clip filmstrip

Horizontal scrollable strip of clip tiles. Each tile is a 16:9
thumbnail (extracted server-side from the clip's midpoint) plus the
clip index, duration, and badge stack (event type, effects count,
caption count).

## States

Two visual states per tile, independent and composable:

| State | What it means | Visual |
|---|---|---|
| **Selected** | User clicked this tile to edit it. The right-side detail panel targets this clip. | Bold `border-2 border-accent` + 5px light-blue ring (`shadow-[0_0_0_5px_rgba(96,165,250,0.4),...]`) + slight `bg-accent/5` |
| **Playing** | Video playhead is currently within this clip's reel range. | Subtle `bg-accent/10` + `border-accent/40` + pulsing accent dot in the top-left overlay |

When a tile is BOTH selected and playing, the selected styling wins
(the playing bg-tint is suppressed in the cn() classes) so the layered
look doesn't get muddy.

## Behaviors

### Click → select + seek

Clicking a tile fires `onSelect(clipId)`. The viewer:
1. Updates `selectedId` state (and writes `#clip=<uuid>` to URL hash).
2. Sets `videoRef.current.currentTime` to the clip's reel-start
   (computed by walking cumulative clip durations).
3. Playback state preserved — playing stays playing, paused stays
   paused; just seeks.

### Vertical wheel → horizontal pan

Hover the filmstrip and use the scroll wheel. Translates `deltaY`
into `scrollLeft += deltaY`.

- Attached via `addEventListener('wheel', ..., { passive: false })`
  in a `useEffect` — React's `onWheel` is passive by default, so
  `preventDefault` would silently fail.
- Skips when `Math.abs(deltaX) > Math.abs(deltaY)` so trackpad
  horizontal two-finger pan still works natively.
- No overflow check — `scrollLeft +=` on a non-overflowing container
  is a no-op, so we don't need to gate it.

### Auto-scroll to playing tile

When `playingId` changes (i.e., video playhead crosses a clip
boundary), the filmstrip pans to center that tile.

- Implementation: `useEffect([playingId], () => container.scrollTo({ left: targetOffset, behavior: 'smooth' }))`
- Manual offset math (`tile.offsetLeft - (container.clientWidth - tile.clientWidth) / 2`)
  instead of `scrollIntoView` — Chrome inconsistently honours
  `inline: "center"`, and `scrollIntoView` can scroll the PAGE when
  the strip isn't the nearest scrollable ancestor.
- Skips when the tile is already centered (within 4px) — avoids
  fighting manual user scroll.

## Backend data

- `caption_segments` + `caption_mode` flow through `GET /clips`
  (the API returns the full segments inline, not just a count).
- Thumbnails live at `<workspace>/compilations/<stem>/_thumbnails/<clip_id>.jpg`.
  Auto-generated by `render_spec`'s thumbnail extraction step;
  backfill old compilations with
  `POST /edit/compile/:id/thumbnails/regenerate`.
- The URL is built client-side from the compilation's `output_path`
  by replacing the trailing `compilation.mp4` with `_thumbnails/`.

## Cache-busting thumbnails

Tile thumbnail URL: `${baseUrl}/${encodeURIComponent(clip.id)}.jpg?v=${cacheBust}`.

`cacheBust` is the journal length (bumps on every successful
mutation). After an edit, thumbnails re-fetch even though the
filename stayed the same.

## Code path

- Component: `src/components/ClipFilmstrip.tsx`

  `#clip=<uuid>`

## Future improvements

- **Drag to reorder** — reorder clips by dragging a tile. Calls the
  existing `POST /edit/compile/:id/reorder` endpoint with a custom
  order (would need a new mutator that accepts an explicit clip-id
  sequence rather than a sort mode).
- **Right-click context menu** — quick actions per tile (remove,
  flip caption mode, set intro before this clip).
- **Tile resize** — let the user shrink tiles for 20+ clip reels to
  fit more on screen.
- **Virtualization** — only needed at 50+ clips. `react-virtual` would
  drop in easily since tiles have fixed width.
